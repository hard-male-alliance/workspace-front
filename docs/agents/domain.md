# Domain Docs

This repository uses a single domain context.

Before exploring domain behavior, read `CONTEXT.md` at the repository root when it exists and relevant architectural decisions under `docs/adr/` when that directory exists. If either location is absent, proceed without inventing missing domain documentation.

Use canonical terms from `CONTEXT.md` in tests, plans, issues, and implementation. Surface any conflict with an existing ADR instead of silently overriding it.
