# 简历三栏工作台设计

## 背景与目标

简历页需要从“内容编辑 + PDF 预览 + 可折叠助手抽屉”调整为常驻的桌面三窗口工作台。用户在 AI 对话中通过自然语言要求生成或修改简历；AI 从有限的契约板块类型中选择合适结构，生成正文，并直接同步到内容编辑与 PDF 预览。用户仍可手动编辑、删除和排序板块，并可撤销每次 AI 修改。

本次仅改造简历页。其他页面不在范围内，移动端现有交互保持不变。

## 已确认的产品决策

- 桌面端从左到右为：AI 对话、内容编辑、PDF 预览。
- 三栏首次进入时全部展开且等宽。
- 三个窗口标题栏始终显示在工作区顶部；收起和展开控件只放在标题栏中。
- 三栏可任意组合展开或收起，也允许全部收起。
- 全部收起后只保留标题栏，工作内容区为空。
- 展开的相邻窗口之间提供可拖动分隔线，交互类似 VS Code。
- 布局调整只在当前页面会话内有效；刷新后恢复三栏等宽并全部展开。
- 内容编辑区默认纵向展示所有板块，点击板块后进入聚焦编辑状态。
- 用户可修改标题和正文、删除板块、拖拽排序板块；新增板块通过 AI 对话完成。
- 页面不提供“生成简历”按钮。用户用自然语言要求 AI 生成简历。
- AI 首次生成和后续修改都直接同步，不经过逐条审批。
- 每次 AI 修改都提供“撤销本次修改”。
- 页面不展示 AI 使用的知识库来源、引用片段或引用提示。
- AI 暂时从现有契约的全部 11 类板块中选择：职业摘要、工作经历、教育经历、项目经历、专业技能、论文发表、荣誉奖项、资格证书、语言能力、志愿经历、自定义板块。
- PDF 窗口标题栏提供模板快速切换；详细设置继续使用现有模板设置页。
- 必须通过稳定 Gateway 为后端接入做准备，不在页面中实现或猜测 transport。

## 信息架构与布局

### 固定窗口标题栏

简历工作区内部由固定标题栏和可变内容区组成。标题栏始终分成三个等宽段，分别对应 AI 对话、内容编辑和 PDF 预览。每段包含窗口名称、适用的简短状态及收起或展开按钮。

标题栏不随内容窗口宽度拖动，避免窗口收起后失去恢复入口。内容窗口按当前展开组合占满可用空间：首次全部展开时等宽；收起窗口后，其余窗口扩展；重新展开时按照当前会话最近的有效比例归一化恢复。全部收起时不渲染内容窗口或分隔线。

### 可调整内容区

内容区使用桌面多面板布局。仅在两个相邻可见窗口之间渲染分隔线。拖动分隔线调整两侧窗口宽度，并设置合理最小宽度，防止输入框、编辑器或 PDF 纸张失去可用性。分隔线使用 `role="separator"`、方向和当前值等无障碍属性，并支持方向键微调。

本次不持久化布局到 localStorage 或后端。路由重新挂载或页面刷新后恢复初始布局。

### 响应式边界

桌面断点使用新的三窗口布局。小屏幕继续沿用现有移动端面板切换和助手抽屉，不在本次任务中重新设计。桌面 CSS 和移动端 CSS 必须隔离，避免三栏改造破坏既有紧凑布局。

## 窗口设计

### AI 对话

AI 窗口展示已有消息、本地发送状态和底部输入区，不提供额外的生成按钮。页面向 Gateway 发送用户自然语言，且只消费结构化的领域结果，不解析 AI 文本来推断简历变更。

产生简历变更的助手消息显示变更摘要与“撤销本次修改”。撤销采用单步语义：只有最近一次尚未撤销且其后未发生用户编辑的 AI 变更可撤销；新的 AI 变更或用户编辑发生后，更早的撤销控件变为不可用，避免恢复旧快照时覆盖更新内容。撤销通过变更标识调用 Gateway，不直接在组件中猜测反向操作。知识库检索发生在未来后端或当前 Mock adapter 内部，页面不展示来源信息。

当前 Mock 应至少覆盖：

- 用户请求生成简历后返回一组合法板块与正文。
- 用户请求修改后返回更新后的简历投影。
- AI 修改立即同步到内容编辑和 PDF 预览。
- 每次 AI 修改返回可撤销的变更标识。
- 撤销成功后恢复变更前的简历投影，并更新消息状态。
- 空输入、进行中重复提交、失败和重试状态。

Mock 行为必须明确标注，不宣称已调用真实模型、知识库或服务端。

### 内容编辑

板块以连续文档结构纵向排列，使用留白和分隔线建立层级，避免嵌套卡片。默认状态显示标题和正文摘要；点击后进入聚焦编辑，展示标题输入、正文编辑器和既有格式工具。

用户可以：

- 修改当前板块标题和正文。
- 删除板块。
- 拖拽调整板块顺序。
- 通过上移和下移按钮完成等价的键盘操作。

本次不提供手动新增入口。新增板块只能来自 AI 对话，并且必须属于 `UiResumeSectionKind` 的既有范围。排序、删除与文本编辑都要立即更新 PDF 视觉投影。未来持久化时仍通过 Gateway 提交领域意图，页面不直接构造 REST DTO。

### PDF 预览

PDF 窗口继续使用现有语义纸张预览。标题栏显示当前模板，并提供模板快速选择及进入现有模板设置页的入口。模板和内容变化立即驱动本地视觉预览，但文案必须说明它是本地 Mock 预览，不得表示已经生成正式 PDF 或保存到服务器。

真实 PDF RenderArtifact、PdfSourceMap、签名 URL、渲染 Job 状态和错误诊断仍由未来 adapter 接入。

## 领域边界与数据流

调用链保持为：

```text
ResumeEditorPage
-> ResumeGateway
-> MockResumeGateway（当前）或 HTTP/SSE adapter（未来）
-> 当前项目后端
```

`ResumeGateway` 应以业务意图表达能力，而不是暴露通用 `request(path, body)`：

- 加载简历工作区。
- 发送简历助手消息。
- 撤销一次 AI 简历变更。
- 提交用户的板块编辑。
- 调整板块顺序。
- 删除板块。
- 切换简历模板。

具体方法签名应复用现有领域模型，并为操作输入、操作结果、变更标识和可撤销状态增加最小必要类型。页面消费适合展示的 `Ui*` 投影；未来 transport DTO、运行时校验和字段映射留在 infrastructure adapter 中。

AI 操作的领域结果至少包含：助手消息、最新简历投影、是否发生简历变更、可选变更标识以及是否可撤销。Mock 和真实 adapter 必须实现同一个 Gateway 接口。

## 后端接入预留

当前契约尚未冻结简历 AI 对话和生成流程的 method、path、DTO、状态码、鉴权、幂等、取消及流式事件。实现期间不得猜测这些细节。应在 `domain/pending.ts` 补充：

- 简历助手消息提交与会话关联方式。
- AI 生成或修改返回完整快照还是操作批次。
- 变更标识与撤销语义。
- Agent Run 与 ResumeOperationBatch 的绑定方式。
- SSE 的事件类型、顺序、结束标志、重连、去重和取消语义。
- 知识库检索授权由受信后端完成，产品页面不得构造可信身份头。
- 模板切换、ETag/If-Match 和冲突处理。
- PDF 渲染任务和最终 artifact 获取方式。

未来 HTTP/SSE adapter 负责响应状态与 content type 检查、`application/problem+json` 转换、未知数据运行时校验、DTO 到领域模型映射、事件跨 chunk 解析、按 event ID 去重、AbortController 清理以及恢复策略。页面调用方式不应因此变化。

## 状态、失败与恢复

- 初始加载覆盖 loading、ready、empty 和 error。
- AI 发送期间禁用重复提交并显示进行中状态。
- AI 失败时保留现有简历，显示原因和重试入口。
- AI 结果只有在 Gateway 成功返回有效领域结果后才替换当前投影。
- 撤销失败时保留当前简历，并恢复撤销按钮的可操作状态。
- 模板切换、删除和排序失败时恢复操作前投影。
- 组件卸载或路由切换时取消仍在进行的操作；Mock 的生命周期应与未来 AbortController 兼容。
- 用户本地编辑与未来服务端 revision 冲突时不得静默覆盖；真实策略等待 ETag/If-Match 契约冻结。

## 可访问性与交互规范

- 所有图标按钮提供可读的 `aria-label`。
- 窗口标题栏按钮暴露展开状态。
- 分隔线支持鼠标、键盘与清晰焦点状态。
- 拖拽排序同时提供上移、下移操作，不把拖拽作为唯一方式。
- 输入、删除、失败和撤销结果通过适当的 live region 告知辅助技术。
- 交互目标尺寸、文字对比度和焦点可见性沿用现有设计令牌并满足项目标准。
- 动效只用于窗口状态和操作反馈，并尊重 `prefers-reduced-motion`。

## 组件划分

为避免继续扩大单个页面组件，实现在简历功能目录内拆分以下聚焦组件：

- `ResumeWorkspaceChrome`：固定标题栏、窗口显隐和面板布局。
- `ResizableResumePanels`：可见窗口、分隔线和会话内宽度。
- `ResumeAssistantPanel`：消息、发送、变更摘要和撤销。
- `ResumeSectionsEditor`：板块列表、聚焦编辑、删除和排序。
- `ResumePreviewPanel`：模板快速选择和语义 PDF 预览。

路由页继续负责加载资源与组合状态。组件不得依赖具体 Mock 类或 transport。

## 测试与验收

### 自动化测试

- 三栏首次全部展开且等宽。
- 每栏可独立收起和展开。
- 任意组合及全部收起状态正确。
- 分隔线调整宽度，键盘操作可用，刷新或重新挂载后重置。
- AI Mock 生成会创建合法板块并同步两个窗口。
- AI Mock 修改直接生效，且可撤销对应变更。
- AI 操作失败不覆盖已有简历。
- 板块聚焦编辑、删除、拖拽排序和键盘排序。
- 模板快速切换更新预览，并保留详细设置路由。
- 页面不展示知识来源或引用。
- 中英文可访问名称正确。
- 移动端既有面板行为不回归。
- Mock gateway 与新增领域端口的契约测试。

## 非目标

- 不实现或猜测真实 HTTP、SSE、WebSocket 或 WebRTC 协议。
- 不让浏览器或 Electron renderer 直接调用外部模型服务。
- 不实现真实 PDF 下载、签名 URL 或 source map。
- 不改变其他业务页面。
- 不重新设计移动端简历体验。
- 不把 Mock 数据散落在页面组件中。
- 不把布局偏好持久化到本地或后端。
